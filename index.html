1

GitHubリポジトリを作成する

GitHubアカウントが必要です。持っていない場合は github.com で無料登録してください。

1-1. 新しいリポジトリを作成

  1. github.com/new を開く
  2. Repository name: ai-news-collector
  3. Private を選択（APIキーを含むため必須）
  4. 「Create repository」をクリック

1-2. 必要なファイルを準備する

以下の5つのファイルを作成してリポジトリにアップロードします。

ai-news-collector/
├── CLAUDE.md　　　← ★設定ファイル（URLやルールを管理）
├── collect.py　　　← メイン処理スクリプト
├── requirements.txt　← 必要ライブラリ
├── .gitignore
└── .github/
　　　└── workflows/
　　　　　　└── daily.yml　← 定時実行設定

1-3. コマンドラインでプッシュする場合

git init
git add .
git commit -m "initial commit"
git remote add origin https://github.com/あなたのユーザー名/ai-news-collector.git
git branch -M main
git push -u origin main
2

Anthropic APIキーを取得・登録する

2-1. APIキーを取得

  1. console.anthropic.com にアクセス
  2. アカウント登録またはログイン
  3. 「API Keys」→「Create Key」でAPIキーを生成
  4. sk-ant-... で始まる文字列をコピー
APIキーは一度しか表示されません。必ずコピーしてメモしてください。

2-2. GitHub Secretsに登録

  1. GitHubリポジトリの Settings を開く
  2. 左メニュー「Secrets and variables」→「Actions」
  3. 「New repository secret」をクリック
  4. 以下を入力して「Add secret」
    • Name: ANTHROPIC_API_KEY
    • Value: 取得したAPIキー
3

Google Sheets APIを設定する

重要： 必ず個人のGmailアカウント（@gmail.com）でGoogle Cloud Consoleにログインしてください。会社・学校アカウントではセキュリティポリシーによりキー作成がブロックされる場合があります。

3-1. Google Cloudプロジェクトを作成

  1. console.cloud.google.com/projectcreate を開く
  2. プロジェクト名: ai-news-collector
  3. 「場所」欄が「組織なし」になっていることを確認（なっていない場合は「参照」→「組織なし」を選択）
  4. 「作成」をクリック

3-2. APIを有効化

  1. 作成したプロジェクトを選択した状態で「APIとサービス」→「ライブラリ」を開く
  2. 「Google Sheets API」を検索して有効化
  3. 「Google Drive API」を検索して有効化

3-3. サービスアカウントを作成

  1. 「APIとサービス」→「認証情報」を開く
  2. 「サービスアカウントを作成」をクリック
  3. サービスアカウント名: ai-news-collector を入力
  4. 「作成して続行」→権限はスキップ→「完了」

3-4. JSONキーをダウンロード

  1. 作成したサービスアカウントをクリック
  2. 「キー」タブ → 「キーを追加」→「新しいキーを作成」
  3. 「JSON」を選択して「作成」
  4. JSONファイルが自動でダウンロードされる
このJSONファイルは絶対にGitHubにアップロードしないでください。.gitignoreで除外されています。

3-5. JSONをGitHub Secretsに登録

  1. ダウンロードしたJSONファイルをテキストエディタで開く
  2. 中身を全てコピー
  3. GitHub → Settings → Secrets → 「New repository secret」
    • Name: GOOGLE_CREDENTIALS_JSON
    • Value: JSONの中身を貼り付け

3-6. スプレッドシートを作成・共有

  1. sheets.google.com で新しいスプレッドシートを作成
  2. URLの spreadsheets/d/★ここ★/edit の部分（スプレッドシートID）をコピー
  3. GitHub Secrets に追加
    • Name: SPREADSHEET_ID
    • Value: コピーしたID
  4. スプレッドシートの「共有」ボタンをクリック
  5. JSONファイル内の client_email の値（メールアドレス）を「編集者」として追加して共有
4

動作確認（手動実行）

  1. GitHubリポジトリの「Actions」タブを開く
  2. 左側の「毎日AI情報収集」をクリック
  3. 右側の「Run workflow」→「Run workflow」をクリック
  4. 実行中は黄色の丸、完了後に緑のチェックマークが表示される
緑のチェックマークが表示されればセットアップ完了！
以後は毎朝8時（日本時間）に自動実行されます。

実行結果の確認場所

  • Markdownファイル：リポジトリの summaries/ フォルダ内に summary_YYYY-MM-DD.md が生成されます
  • Googleスプレッドシート：日付名のシートが自動作成され、記事一覧が書き込まれます
5

設定をカスタマイズする

設定変更は CLAUDE.md を編集するだけです。スクリプトを触る必要はありません。

監視URLを追加・変更する

## 監視URL セクションに
- https://新しいURL を追記

収集するキーワードを変える

## 収集ルール セクションを編集。例：「セキュリティ関連のみ」など

要約スタイルを変える

## 要約スタイル セクションを編集。自分の観点・興味に合わせて調整可能

記事数を絞る

## 収集ルール に「重要度が高い記事を〇件のみに絞ること」と追記

実行時間を変更する

.github/workflows/daily.yml の cron を変更（UTC時間で設定・JST = UTC+9）

# 毎朝8時（JST）= UTC 23時
- cron: '0 23 * * *'

# 毎朝6時（JST）に変更する場合 = UTC 21時
- cron: '0 21 * * *'
